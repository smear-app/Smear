Project: Smear — climbing progression gamification

Purpose
-------
Smear helps rock climbers track, quantify, and gamify their route progression across gyms and outdoor crags. Users should be able to log routes (regardless of gym), record rich metrics, visualize progress, and compete/challenge themselves and friends.

Core features (MVP)
-------------------
- Sign up / authentication
- Log a climb with attributes: grade, style (boulder/toprope/lead), attempt type, attempts, send boolean, date, notes
- Profile with history and simple analytics
- Basic gamification: XP, levels, badges

Stack
-----
- Backend: Python + FastAPI
- Database: MongoDB
- Frontend: React (TypeScript recommended)

How to use the docs folder
--------------------------
- Read `ARCHITECTURE.md` for the system layout
- Use `DATA_MODEL.md` as the source of truth for document schemas
- Use `API.md` to scaffold FastAPI endpoints
- Use `FRONTEND.md` for UI structure and UX decisions
